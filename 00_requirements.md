# 毕设需求冻结（第一版）

## 1) 任务定义
- 方向：多模态实体对齐的跨域可迁移性（TMMEA-DA 主线）
- 目标：在跨语言（DBP15K）与跨图谱（FB15K-DB15K / FB15K-YAGO15K）设置下提升迁移性能

## 2) 研究问题（RQ）
- RQ1：跨语言与跨图谱场景中，哪些域差异是性能退化主因？
- RQ2：域对齐与多源选择模块能否稳定降低负迁移？
- RQ3：在模态缺失/噪声下，鲁棒融合是否仍保持优势？

## 3) 可检验假设（H）
- H1：引入域对齐损失可显著提升目标域 Hits@1 / MRR。
- H2：多源域选择比单源迁移更稳健（多 seed 方差更小）。
- H3：缺失感知融合在高缺失率下相对基线退化更小。

## 4) 统一指标
- 主指标：Hits@1, Hits@10, MRR
- 辅指标：MR, 训练时间, 显存峰值
- 稳定性：关键配置至少 5 seeds，报告 mean ± std

## 5) 数据与设定（待你持续补充）
- DBP15K：ZH-EN / JA-EN / FR-EN（默认 train_ratio=0.3）
- 跨图谱：FB15K-DB15K / FB15K-YAGO15K
- 缺失策略：mask + missing embedding（后续在 config 固化）

## 6) 算力与约束
- GPU：RTX 3060 Laptop GPU
- CUDA：12.6（驱动上限 13.1）
- 主环境：conda `bysj-main`（Python 3.10 + torch cu126）

## 7) 里程碑（示例）
- M1：环境+数据规范冻结
- M2：基线可复现
- M3：方法 MVP 可收敛
- M4：消融/稳定性完成
- M5：中期/终稿材料完备
